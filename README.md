## What it does

Most PS tools show you current margin. This app forecasts *future* margin — using hours burn rate, schedule velocity, billable ratio, and contract type to predict where a project will land at completion, **before it gets there**.

### Interactive AI Custom Queries (Claude-Powered)
- **Inline Custom Queries**: Users can type custom questions or use pre-populated quick action buttons (e.g. *Actions to protect margin*, *Model overrun*, *Draft escalation email*, and *Get action plan*) to query Claude directly inside project detail cards. Responses are rendered inline.
- **Project-Context Enrichment**: Every custom query automatically aggregates and prepends the corresponding project's real-time financials and indicators. This allows Claude to provide specific, numbers-driven feedback without user retyping.
- **System Constraints & Topic Safety**: Enforces system constraints in `callClaude.js` to ensure responses remain strictly focused on project metrics and delivery health (politely declining off-topic queries like food recipes or general knowledge).

### Scenario Graphical Comparisons
- **Hours comparison chart**: Dynamically visualizes Budgeted Hours vs. Estimate at Completion (EAC) vs. Allocated Hours (worst-case allocation).
- **Margin comparison chart**: Dynamically visualizes Target Margin vs. Forecast Margin (EAC) vs. Allocated Margin (margin if all allocated hours are burned).
- **Interactive Visual Indicators**: Charts dynamically scale, color-code (red for overrun, amber for alert, green for safe), and display percentage indicators.

### Three layers

| Layer | What it does |
|---|---|
| **Portfolio** | Sortable table of all 27 projects with EAC, forecast margin %, and risk classification. Includes interactive charts and custom query inputs. |
| **Governance** | Risk notifications + 4-action playbooks for Critical, High, and Medium risk projects. Includes inline project-specific Claude query submission. |
| **Notifications** | Alert feed for Critical and High risk projects with one-click action prompts and inline Claude query input. |

### Risk model

| Level | Trigger |
|---|---|
| 🔴 Critical | Allocated hours > 3× budget, OR EAC overrun > 20% |
| 🟠 High risk | Forecast margin < target, OR schedule velocity gap > 25% |
| 🟡 Medium | Hour creep 8–20%, velocity gap 12–25%, billable ratio < 70%, or data gap |
| 🟢 On track | All checks pass |
| ⚪ N/A | NON_BILLABLE contract — excluded from margin model |

## Local development

### 1. Clone and install

```bash
git clone https://github.com/smitsaurabh/margin-slip-detector
cd margin-slip-detector
npm install
```

### 2. Configure local installation params

Edit `.rli/installation.json` to set your financial model parameters:

```json
{
  "appInstallationId": 1,
  "appId": 1,
  "versionId": 1,
  "iparams": {
    "billableRate": 72,
    "costRate": 51,
    "targetMargin": 25
  },
  "secureParams": {},
  "productIdentifier": "ROCKETLANE",
  "productAccountId": 1,
  "createdAt": 1,
  "isDeleted": false,
  "disabled": false,
  "isLatest": true
}
```

**Recommended values** (derived from your Rocketlane actuals):

| Field | Value | How derived |
|---|---|---|
| `billableRate` | `72` | T&M actuals: $43k revenue ÷ 524 billable hours across completed T&M projects |
| `costRate` | `51` | All completed projects: $332k total cost ÷ 6,420 tracked hours |
| `targetMargin` | `25` | TSIA PS industry benchmark floor (25–35%) |

### 3. Start the local dev server

```bash
npm start
# or
rli dev
```

This starts a local development server. Open your browser to the URL shown in the terminal (typically `http://localhost:10001`).

The dev server:
- Hot-reloads the widget HTML when you save changes
- Runs the server action (`getMarginForecast`) locally against mock or live data
- Shows validation errors in the terminal in real time

### 4. Connect to live Rocketlane data (optional)

To hit real Rocketlane API data during local dev, set your API key as an environment variable:

```bash
export ROCKETLANE_API_KEY=your_api_key_here
npm start
```

Get your API key from: **Rocketlane → Settings → Developer → API Keys → Generate Key**

### 5. Edit the widget

The main UI lives in `widgets/margin-slip-detector/index.html`. This is a self-contained HTML file — no build step needed for front-end changes. Edit and save; the dev server reloads automatically.

The server action logic lives in `server-actions/getMarginForecast.js`. This file:
- Paginates through all Rocketlane projects using `r.api.get()`
- Computes EAC, forecast margin, overrun %, velocity gap, and billable ratio for each project
- Classifies each project into a risk level
- Returns a sorted portfolio object to the widget

## Build for deployment

```bash
npm run build
# or
rli build
```

This command:
1. Validates `index.js` — checks all widgets and server actions are correctly defined
2. Bundles the server action into `rli-dist/rli-server.js`
3. Generates `rli-dist/deploy.json` — the deployment manifest Rocketlane requires
4. Processes widget HTML templates (injects the Rocketlane SDK script tag)
5. Packages everything into `app.zip`

Successful output looks like:

```
Installation fields are valid
App is valid
🚀 Building the app
Server files bundled
Deploy config generated
App packed
Build completed in ~8s
```

The upload-ready file is `app.zip` in the project root.

## Configuration reference

The app has three installation parameters configurable at install time and adjustable later in **Settings → Apps → Margin Slip Detector → Configure**:

| Parameter | Default | Description |
|---|---|---|
| `billableRate` | `72` | Your blended hourly billable rate in USD. Used to compute projected revenue for T&M contracts. |
| `costRate` | `51` | Your blended hourly fully-loaded cost rate in USD. Used to compute projected cost (EAC × costRate). |
| `targetMargin` | `25` | Minimum acceptable margin % before the app flags a project as High risk. |

**Tuning tips:**
- If your portfolio skews fixed-fee, the billable rate matters less — the model uses `projectFee` as projected revenue for fixed-fee contracts regardless of hours.
- If you have distinct rate cards for different customer tiers (Platinum vs. Silver), use the weighted average across your active portfolio.
- Tighten `targetMargin` to 30% if you want earlier warnings; loosen to 20% if the current threshold is too noisy.

## How the margin model works

```
EAC (Estimate at Completion)
  = (trackedHours / progressPct) × 100         [if time has been logged]
  = budgetedHours                               [if no time logged yet]

Projected cost = EAC × costRate

Projected revenue
  = projectFee                                  [FIXED_FEE contracts]
  = EAC × billableRate                          [TIME_AND_MATERIAL contracts]
  = 0 / excluded                                [NON_BILLABLE contracts]

Forecast margin % = (projectedRevenue − projectedCost) / projectedRevenue × 100

Hours overrun % = (EAC − budgetedHours) / budgetedHours × 100

Velocity gap = (ageDays / plannedDays × 100) − progressPct
```

A positive velocity gap means time is passing faster than work is completing — the project is running behind schedule.

## Troubleshooting

### "rli: command not found"

```bash
npm install -g @rocketlane/rli --ignore-scripts
export PATH="$PATH:$(npm root -g)/../bin"
```

Or use npx: `npx @rocketlane/rli dev`

### "Invalid zip: rli-dist/deploy.json not found"

You uploaded the source code zip instead of the build output. Run `npm run build` first — this generates `app.zip` with the correct structure including `rli-dist/deploy.json`.

### Widget shows "Failed to load data"

- Check that your Rocketlane API key has **read access** to projects
- Verify the app is installed (Settings → Apps → Custom Apps → Margin Slip Detector → Active)
- Try refreshing the widget using the **⟳ Refresh** button
- Check browser console for network errors

### Claude AI brief shows "API unavailable"

The widget calls the Anthropic Claude API directly from the browser. Ensure:
- The Claude API is accessible from your network
- No browser extension is blocking requests to `api.anthropic.com`

### Local dev server shows validation errors

Common causes:
- `index.js` references a widget or server action file that doesn't exist — check file paths
- `installation.json` has invalid JSON — validate with `node -e "require('./.rli/installation.json')"`
- Node.js version below v18 — upgrade with `nvm use 18`
